# Spec Templates

File templates for `/spec new`. Write each to its target path under `specs/`.

A spec is **two files plus an examples directory**: `SPEC.md` (human goal +
agent design) and `STATUS.md` (tracking, run log, handoffs). Nothing else.

<templates>

<template file="specs/AGENTS.md">
# Specs

Specs are private working context and must never be committed. Keep `specs`
ignored in git; this directory may be a symlink to a private per-repo specs
directory.

Each spec lives in `specs/<feature>/` as two files plus `examples/`.

Read order:

1. `STATUS.md` (entry point — phase, next, run log, handoffs)
2. `SPEC.md` (goal + design)
3. `examples/` (runnable verification scripts)

Durable design decisions do NOT live here — they go in committed `docs/adr/`.
The optional domain glossary lives in committed `CONTEXT.md` at the repo root.

Do not manually maintain an index. The project-level `specs/STATUS.md` is
generated by `spec_status.py`; do not edit it by hand.
</template>
