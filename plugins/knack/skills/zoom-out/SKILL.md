I don't know this area of code well. Go up a layer of abstraction. Give me a map
of the relevant modules and their callers — what each one is responsible for and
how they connect — using the project's own vocabulary (the `CONTEXT.md` glossary
if one exists). Diagrams or a short module list are fine; the goal is orientation,
not exhaustive detail.
